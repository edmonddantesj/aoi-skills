# Moltbook Writer Profile System v0.1

## Goal
Moltbook daily drafts must no longer be generated by swapping only the writer name.
Each draft should reflect the selected writer's:
- role axis
- preference axis
- preferred topics
- avoid topics
- tone / voice
- typical angles
- CTA style

Canonical end matter remains standardized, but the body, hook, examples, and CTA must change with the writer.

## Source of truth
- Primary operating DB: `Moltbook Writer Profiles` under Notion page `👥 Aoineco & Co. Squad Dashboard`
- This local document is the durable fallback/spec.

## Core operating rules
1. Writer rotation is based on 12-member roster.
2. Exclude writers whose `Rotation Status` is `cooling` or `used_this_cycle`.
3. `청묘` is kept in the DB for team completeness, but external posting default is `External Safe? = no`.
4. Daily draft generation must consider:
   - rotation fit
   - topic fit
   - freshness / repeat avoidance
   - writer profile alignment
5. The following MUST change when writer changes:
   - topic
   - hook
   - examples
   - argumentative angle
   - CTA
6. The following remain standardized:
   - canonical end matter
   - approval gate
   - proof / archive rules
   - public-safe rules

## Required output format for daily report
- Today’s writer: <name>
- Role axis: ...
- Preference axis: ...
- Why today: rotation + topic fit
- Avoided repeats: ...
- Draft: ...
- Canonical end matter: included

## Review split
- Writer: profile-selected author persona
- Reviewer: assigned separately based on review role
- Default review lenses:
  - 청비: structure / knowledge hygiene
  - 청령: message fit / operating fit
  - 청묘: final assembly / chair-facing polish

## Draft generation flow (target)
1. Scan recent Moltbook / 봇마당 / internal outputs
2. Pull eligible writer candidates from profile system
3. Exclude recent / cooling writers
4. Generate 3 topic candidates using selected writer profile
5. Pick 1 candidate and assemble near-post-ready draft
6. Report to topic 1114 with writer rationale + draft
7. After approval, upload
8. Update writer usage fields in Notion

## Notes

- Do not expose `청묘/Chungmyo/CHUNGMYO` in external posts.
